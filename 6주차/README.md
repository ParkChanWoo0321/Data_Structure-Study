# 알고리즘 구현 과제 (C) 📌

이 레포지토리는 C로 구현한 알고리즘 과제 코드입니다.  
- **탐욕 알고리즘**으로 최소 신장 트리(MST) 구성  
- **백트래킹**으로 스도쿠 풀이

---

## 1) 최소 신장 트리(MST) — Kruskal (Greedy) ✅  
**파일:** `6-1.c`

### 개요
가중치가 있는 그래프에서 모든 정점을 연결하되, **총 가중치 합이 최소가 되도록** 간선을 선택합니다.

### 구현 방식
- **Kruskal 알고리즘**
  1. 모든 간선을 가중치 오름차순으로 정렬(`qsort`)
  2. 정렬된 간선을 순서대로 확인
  3. 사이클이 생기지 않으면 선택(Union-Find로 판별)
  4. 간선을 `V-1`개 선택하면 종료

### 사용된 핵심 자료구조
- **Union-Find(Disjoint Set)**
  - `find()` : 경로 압축(Path Compression) 적용
  - `unionSet()` : 서로 다른 집합이면 합치기

### 출력
- 선택된 MST 간선 목록
- MST 총 가중치

### 시간 복잡도(일반)
- 간선 정렬: **O(E log E)**
- Union-Find: 거의 **O(1)**(역아커만 수준)  
→ 전체: **O(E log E)**

## 2) 스도쿠 풀이 — Backtracking ✅  
**파일:** `6-2.c`

### 개요
빈 칸(0)을 채워 **스도쿠 규칙(가로/세로/3×3)** 을 만족하는 완성 해를 찾습니다.

### 구현 방식
- **백트래킹(재귀 탐색)**
  1. 빈 칸(0)을 찾는다
  2. 1~9를 순서대로 넣어본다
  3. 넣을 수 있으면 다음 칸 재귀 진행
  4. 막히면 되돌아가서(Backtrack) 다른 숫자 시도

### 핵심 함수
- `isValid(row, col, num)`
  - 같은 행/열에 중복이 있는지 검사
  - 해당 3×3 블록에 중복이 있는지 검사
- `solveSudoku()`
  - 빈 칸을 찾아 유효한 숫자를 대입하며 재귀적으로 해결

### 출력
- 해결된 스도쿠 보드 출력  
- 해결 불가능 시 실패 메시지 출력

### 시간 복잡도(일반)
- 최악의 경우 경우의 수가 커질 수 있어 **지수 시간** 가능  
  (입력 난이도에 따라 탐색량이 크게 달라짐)
gcc 6-2.c -o sudoku
./sudoku
