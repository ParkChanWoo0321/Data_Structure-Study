# 📚 학습 내용 요약

## 1. 스택/큐/다항식 (1-1 ~ 1-3)
- 배열 기반 스택(LIFO) / 원형 큐(FIFO)
- 연결리스트 기반 스택/큐(동적 메모리로 크기 제한 제거)
- 다항식 덧셈:
  - 배열(정렬된 항 처리, 동일 지수 합산)
  - 연결리스트(노드 기반 항 저장, 결과 리스트 구성)

---

## 2. 수식 변환/계산 & 미로 탐색 (2-1 ~ 2-4)
- 스택으로 연산자 우선순위 처리(isp/icp), 괄호 처리
- 중위 → 후위 변환 + 후위 계산기(여러 자리 정수 파싱)
- 중위 → 전위 변환 + 전위 계산기(역방향 처리)
- 미로 찾기:
  - 8방향 탐색(move 배열)
  - mark 배열로 방문 체크
  - 스택 기반 DFS로 경로 저장/백트래킹

---

## 3. 희소행렬 전치 & 이진트리 (3-1 ~ 3-4)
### 희소행렬
- 3-튜플(term) 배열 표현 `(row, col, value)`
- 단순 전치 vs 빠른 전치 비교
  - 단순 전치: 열 기준 반복 탐색 → 반복이 많아짐
  - 빠른 전치: `rowTerms[]`, `startingPos[]`로 위치 계산 → 효율 개선
- `3-3.c`에서 실제 연산을 count로 세어 비교

### 이진트리
- 포인터 기반 연결 표현으로 트리 구성
- 순회 3종 재귀 구현
  - 전위: Root → Left → Right
  - 중위: Left → Root → Right
  - 후위: Left → Right → Root

---

## 4. 정렬 & 해싱 (4-1 ~ 4-3)
### Quick Sort (교재 7.6) + 개선
- 첫 원소 pivot 기반 파티션, 재귀 분할 정렬
- 시간 복잡도:
  - 평균: `O(n log n)`
  - 최악: `O(n^2)` (pivot이 한쪽으로 치우치는 경우)
- Hybrid Quick Sort:
  - 작은 구간은 Insertion Sort로 처리(실측 성능 개선)
  - 평균 `O(n log n)` 유지, 상수 시간 개선 목적

### Heap Sort (교재 7.12, 7.13)
- 최대 힙 구성 후 루트-마지막 swap + adjust 반복
- 최선/평균/최악 모두 `O(n log n)` (입력 영향 적음)

### Hash Table (선형 조사법)
- 문자열 키 → 정수 키 변환(`stringToKey`: ASCII 합)
- 충돌 시 `(i+1) % B`로 탐색(Linear Probing)
- 평균 `O(1)`, 충돌 몰리면 최악 `O(n)` 가능(클러스터링)

---

## 5. 분할정복 & DP
### 분할정복: 최대 구간 합
- 좌/우/교차(cross) 최대값 비교로 결과 결정
- 시간 복잡도: `T(n)=2T(n/2)+O(n)` → `O(n log n)`

### DP: Floyd-Warshall (모든 쌍 최단경로)
- 점화식: `dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`
- 시간: `O(n^3)`, 공간: `O(n^2)`
- `INF`로 경로 없음 표현

---

## 6. Greedy & Backtracking (6-1 ~ 6-2)
### MST: Kruskal (Greedy)
- 간선 정렬 `O(E log E)`
- Union-Find로 사이클 판별(경로 압축)
- 전체 복잡도: `O(E log E)`

### Sudoku: Backtracking
- 빈칸(0) 찾기 → 1~9 시도 → 유효하면 재귀 → 실패 시 되돌리기
- 최악의 경우 지수 시간(입력 난이도에 따라 탐색량 변화)
