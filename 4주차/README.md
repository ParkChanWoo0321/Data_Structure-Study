# 자료구조 구현 실습 정리 (C) 📌

교재 예제 기반으로 **퀵 소트 / 힙 소트 / 해시 테이블(선형 조사법)** 을 구현하고, 퀵 소트는 **시간 복잡도 분석 + 개선 버전(하이브리드)** 까지 적용했습니다.

---

## 1) Quick Sort 구현 + 시간 복잡도 분석 + 개선 버전

### ✅ 1.1 교재 버전 Quick Sort (프로그램 7.6) — `4-1.c`
- **파티션 방식**: 첫 원소를 pivot으로 선택
- `i`는 pivot보다 큰 값 탐색, `j`는 pivot보다 작은 값 탐색 후 swap
- 분할 완료 후 재귀적으로 좌/우 구간 정렬
- `clock()`으로 실행 시간(ms) 측정

**시간 복잡도**
- 평균/일반: **O(n log n)**  
- 최악(이미 정렬된 입력, pivot이 항상 한쪽으로 치우침): **O(n²)**  
- 추가 공간: 재귀 호출 스택 **O(log n)**(평균), 최악 **O(n)**

---

### ✅ 1.2 개선 Quick Sort (Hybrid: Quick + Insertion) — `4-1.c` 내부(개선 버전)
- **개선 아이디어**: 작은 구간에서는 Quick Sort보다 **Insertion Sort가 빠른 경우가 많음**
- `THRESHOLD` 이하의 부분 배열은 **Insertion Sort로 처리**
- 큰 구간은 Quick Sort 유지 → 전체 성능 개선(특히 작은 구간이 많아지는 후반부)

**복잡도 관점**
- 평균적으로 Quick Sort의 **O(n log n)** 유지
- 작은 구간 처리 비용을 줄여 **상수 시간(실측 성능) 개선**
- 최악 케이스 자체는 pivot 전략이 같으면 여전히 **O(n²)** 가능  
  (추가 개선을 하려면 median-of-three pivot, random pivot 등의 전략이 필요)

## 2) Heap Sort 구현 (프로그램 7.12, 7.13) — `4-2.c`

- 배열을 **최대 힙(Max Heap)** 으로 만든 뒤
- 루트(최댓값)와 마지막 원소를 swap하고 heap 재조정(`adjust`)을 반복
- 구현 특성상 인덱스를 `1`부터 사용하도록 `list[11]`로 옮겨 담아 처리

**시간 복잡도**
- 힙 구성: **O(n)**
- 정렬 과정: **O(n log n)**
- 최선/평균/최악 모두 **O(n log n)** (입력 상태에 크게 영향 없음)

**특징**
- Quick Sort보다 최악 케이스에 안전
- 일반적으로 Quick Sort보다 상수항이 커서 평균 성능은 불리할 수 있음

## 3) 해시 테이블 구현 (C 키워드 저장: 삽입/검색/삭제) — `4-3.c`

요구 조건:
- 키: 문자열(C 키워드 등)
- 버킷 크기: 자유(여기서는 `B = 11`)
- 슬롯: 1
- 해시 함수: 제산 함수(`k % B`)
- 오버플로우 처리: **선형 조사법(Linear Probing)**

### 핵심 구성
- `stringToKey()` : 문자열을 정수 키로 변환(문자 ASCII 합)
- `insert()` : 충돌 시 `(i+1) % B`로 이동하며 빈 슬롯 탐색
- `search()` : 홈 버킷부터 선형 조사로 키 탐색
- `deleteKey()` : 삭제 처리(더미/표시 방식 사용 의도)
- `printHashTable()` : 현재 테이블 상태 출력

**시간 복잡도(평균)**
- 삽입/검색/삭제 평균: **O(1)**
- 최악(충돌이 몰리는 경우): **O(n)**

**특징**
- 선형 조사법은 구현이 간단하지만
  - **클러스터링(1차 군집)** 이 생기면 성능이 급격히 떨어질 수 있음
