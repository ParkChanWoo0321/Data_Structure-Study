# 📚 자료구조 실습 3 (C) — 희소행렬 전치 & 이진트리 순회

이 레포지토리는 자료구조 교재 예제를 기반으로 **희소행렬(Sparse Matrix) 전치**와 **이진트리(Binary Tree) 순회**를 직접 구현하면서 공부한 내용을 정리한 실습 코드 모음입니다 ✍️  
단순 구현에서 끝나지 않고, 전치 알고리즘 2종에 대해 **연산 횟수를 직접 세어(Count) 시간복잡도 관점으로 비교**도 해봤습니다 ✅

## 1) 🧾 `3-1.c` — 희소행렬 전치 (프로그램 2.8)

### ✅ 공부한 내용
- 희소행렬을 **(row, col, value)** 형태의 `term` 배열로 저장하는 방식 학습
- 전치(transpose) 시 **row/col을 바꿔서** 결과 배열에 넣는 과정을 구현

### 🔍 핵심 포인트
- `a[0]`에는 메타 정보 저장  
  - `a[0].row` = 전체 행 개수  
  - `a[0].col` = 전체 열 개수  
  - `a[0].value` = 0이 아닌 값(항)의 개수
- 단순 전치 방식은 **열을 기준으로 전체 항을 계속 검사**하는 구조라 반복이 많아질 수 있음

---

## 2) ⚡ `3-2.c` — 희소행렬 빠른 전치 (프로그램 2.9)

### ✅ 공부한 내용
- 단순 전치보다 효율적인 **Fast Transpose** 방식 구현
- 전치 결과를 더 빠르게 만들기 위해 **보조 배열을 이용한 위치 계산**을 학습

### 🔍 핵심 포인트
- `rowTerms[]` : 각 열(column)에 몇 개의 항이 있는지 카운트
- `startingPos[]` : 전치 결과 배열에서 각 열이 시작되는 인덱스 계산
- 한 번의 순회로 바로 위치를 찾아 저장 → 불필요한 중첩 탐색이 줄어듦 🚀

---

## 3) ⏱️ `3-3.c` — 시간복잡도 비교(연산 횟수 Count)

### ✅ 공부한 내용
- “빅오만 외우기”가 아니라, **실제로 반복문에서 수행되는 연산을 count로 측정**해 비교해봄 🧠

### 포함 내용
#### 3.1 단순 전치(transpose) count 측정
- 중첩 반복문 내부에서 비교가 실행되는 횟수를 누적하여 출력
- 출력 형태 예시:  
  - `O(n) : count (col x value)` 형태로 반복 구조를 체감

#### 3.2 빠른 전치(fastTranspose) count 측정
- 열 초기화, 열별 항 카운트, startingPos 계산, 실제 배치 과정에서 count 누적
- 전체적으로 `numCols + numTerms` 중심 흐름이라 더 효율적임을 확인 ✅

---

## 4) 🌳 `3-4.c` — 이진트리 연결 표현 + 순회(중/전/후)

### ✅ 공부한 내용
- 교재 그림 5.16 형태의 트리를 **포인터 기반 연결 표현(Linked Representation)**으로 직접 구성
- 재귀 함수로 순회 3가지를 각각 구현하면서 방문 순서를 확실히 정리함 📌

### 🔁 구현한 순회 종류
- **중위순회(Inorder)**: `Left → Root → Right`
- **전위순회(Preorder)**: `Root → Left → Right`
- **후위순회(Postorder)**: `Left → Right → Root`
