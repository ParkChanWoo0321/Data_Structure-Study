# 자료구조 실습 코드 모음 (Sparse Matrix Transpose & Binary Tree Traversal)

이 저장소는 자료구조 수업/교재 예제를 기반으로 **희소행렬 전치 알고리즘**과 **이진트리 순회 알고리즘**을 직접 구현하고, 일부는 **연산 횟수를 세어 시간복잡도를 비교**한 실습 코드들로 구성되어 있습니다.

---

## 1) 희소행렬 전치 (교재 프로그램 2.8)

### 목표
- 희소행렬을 `(row, col, value)` 형태의 **term 배열(3-튜플)**로 저장한 뒤,
- 전치 행렬을 동일한 표현으로 생성하는 전치 알고리즘을 구현했습니다.

### 핵심 구현 포인트
- `a[0]` : 원본 행렬 정보(행/열/값 개수)를 저장하는 헤더
- `a[1..n]` : 0이 아닌 항(Non-zero term)들의 `(row, col, value)` 저장
- 전치 결과는 `b[]`에 저장하며, 원소의 `row`/`col`을 바꿔 배치합니다.

---

## 2) 희소행렬 빠른 전치 (교재 프로그램 2.9)

### 목표
- 1번 전치 알고리즘보다 효율적인 **Fast Transpose**를 구현했습니다.

### 핵심 구현 포인트
- `rowTerms[col]` : 각 열(column)에 존재하는 항의 개수 카운트
- `startingPos[col]` : 전치 행렬에서 해당 열이 시작되는 위치 계산
- 한 번의 스캔으로 `b[]`에 바로 저장하여 불필요한 반복을 줄입니다.

---

## 3) 전치 알고리즘 시간복잡도 비교(연산 횟수 측정)

### 3.1 일반 전치(transpose) 연산 횟수 측정
- 중첩 반복문 내부 비교 횟수를 `count`로 누적
- 실행 결과로 `(col x term)` 형태에 가까운 반복 구조를 확인할 수 있습니다.

### 3.2 빠른 전치(fastTranspose) 연산 횟수 측정
- 열 카운트/시작 위치 계산/배치 과정을 각각 카운트로 측정
- 전체적으로 `numCols + numTerms` 중심으로 동작하는 구조를 확인할 수 있습니다.

> 목적: 단순히 “빅오 표기”만 외우는 것이 아니라, **실제 루프/연산 흐름에서 차이가 어떻게 발생하는지 수치로 확인**하는 실습입니다.

---

## 4) 이진트리 연결 표현 + 순회 구현 (교재 그림 5.16 + 프로그램 5.1~5.3)

### 목표
- 이진트리를 연결 표현(포인터 기반)으로 구성하고
- **중위/전위/후위 순회**를 각각 재귀로 구현했습니다.

### 트리 구성 방식
- `node` 구조체: `data`, `leftChild`, `rightChild`
- `createNode()`로 노드를 생성한 뒤 직접 링크를 연결하여 트리를 구성합니다.

---

### 4.1 중위 순회 (Inorder, 프로그램 5.1)
- 방문 순서: **Left → Root → Right**
- 수식 트리/이진 탐색 트리에서 의미 있는 출력 순서 학습에 사용됩니다.

### 4.2 전위 순회 (Preorder, 프로그램 5.2)
- 방문 순서: **Root → Left → Right**
- 트리 구조를 복사하거나 저장할 때 자주 사용됩니다.

### 4.3 후위 순회 (Postorder, 프로그램 5.3)
- 방문 순서: **Left → Right → Root**
- 트리 삭제(메모리 해제)나 계산 트리 평가에서 자주 사용됩니다.

---

## 정리(학습 포인트)

- 희소행렬을 3-튜플(term) 배열로 표현하는 방식 이해
- 전치 알고리즘 2종 구현 및 **반복 구조 차이** 이해
- 단순 전치 vs 빠른 전치의 **성능 차이를 연산 횟수로 확인**
- 이진트리 연결 표현(포인터 구조) 구성 방법 학습
- 중위/전위/후위 순회의 방문 순서 및 재귀 구현 방식 학습
